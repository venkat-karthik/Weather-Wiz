🌦️ Weather Clone App — Product Requirements Document (PRD)
Version: 1.0
Platform: Replit
Main Language: Python
API: OpenWeatherMap or WeatherAPI (free tier)
Frontend: HTML + CSS + JS (Jinja2 if using Flask)

🧠 1. Purpose
Build a weather app clone inspired by Apple’s iPhone Weather App using only Python for backend and Weather API for real-time data. The goal is to display clean, animated weather dashboards with full hourly/daily info and live radar-like maps.

🎯 2. Core Features
2.1 Home (Current Weather View)
🌡️ Current temperature

🌤️ Weather condition icon + description (e.g., Sunny, Cloudy)

📍 Location name (auto or user-entered)

⏰ Last updated time

🎞️ Background animation based on condition (clouds, rain, etc.)

2.2 Hourly Forecast
Next 24 hours

Temp, icon, precipitation chance

Scrollable or in card format

2.3 7/10 Day Forecast
Day-wise cards:

Max/min temp

Weather icon

Precip chance

2.4 Data Widgets
Each as its own card:

🌬️ Wind speed/direction

💧 Humidity

🌞 UV Index

🌫️ Visibility

🌡️ Feels Like temp

🌀 Pressure

🌙 Moon phase (optional)

🌅 Sunrise/Sunset

🌍 Air quality index (if available in API)

2.5 Search & Multi-city
Search bar to add any city

Save multiple cities (stored in session/local cache)

Switch between them

2.6 Live Map (simplified)
Embed OpenWeather tile-based maps or radar-like image

Layers toggle: temp, clouds, precipitation

⚙️ 3. Technical Stack
3.1 Backend (Python)
Flask (or FastAPI) as backend framework

Requests lib for API calls

Jinja2 templates for rendering pages

OpenWeatherMap API (or WeatherAPI)

3.2 Frontend (within Replit)
HTML5 + CSS3 + JavaScript

Optional: TailwindCSS/Bootstrap for styling

Custom animations using JS/CSS (like raining effect, sun glow, etc.)

3.3 APIs to Use
🔗 https://api.openweathermap.org/data/2.5/weather

🔗 https://api.openweathermap.org/data/2.5/forecast

Optional: https://api.openweathermap.org/data/2.5/air_pollution

Optional: Moon phase data via RapidAPI or manually calculated

📐 4. UI/UX Design
Visuals
Clean, mobile-first layout

Apple-like card design

Dark/light mode toggle

Animated weather backgrounds (CSS/JS-based)

Pages
/ – Current weather

/forecast – Hourly + 7-day

/search – Search bar + city list

/city/<name> – Individual city weather

/map – Map viewer

🧪 5. Functional Requirements
Feature	Description
Fetch Weather	GET request to weather API with city/location
Search City	User enters city name
Multiple Cities	Store cities in session / local cache
Animation Rendering	Use weather condition code to trigger animation
Refresh	Manual or auto-refresh every X minutes
Error Handling	If API fails or city not found
Unit Switching	°C to °F toggle
Localization	Optional, add language support for labels

🚫 6. Out of Scope (for v1)
No native mobile support (not a real iOS/Android app)

No login or user authentication

No in-depth radar maps (just basic embedded tiles)

No backend DB (use session/cache only)

Can display real-time weather for any city.

Hourly and daily forecasts work correctly.

Animations match the weather condition.

Clean, responsive design.

Map renders without errors.

App hosted on Replit, shareable URL.
